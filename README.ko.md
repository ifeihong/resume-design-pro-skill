### 2. AI 콘텐츠 심층 최적화 엔진 -- 진실성 우선

당신의 콘텐츠는**전문적으로 다시 작성**됩니다 -- 당신의 실제 정보에 기반하며, 결코 조작하지 않습니다:

| 최적화 전 | 최적화 후（데이터 제공 시） | 최적화 후（데이터 없음） |
|----------|------------------------|----------------|
| "웹사이트 개발 담당" | "회사 웹사이트 풀스택 개발을 주도하여 전환율 35% 향상" | "회사 웹사이트 풀스택 개발을 주도하여 상당한 비즈니스 영향력 창출" |
| "팀워크에 능숙" | "8명 이상의 교차 기능 팀과 협력하여 3개의 주요 릴리스 성공적 배달" | "교차 기능 팀과 협력하여 여러 주요 릴리스 성공적 배달" |
| "React를 빠르게 학습" | "2주 만에 React 생태계 마스터, 3번째 스프린트에 프로덕션 기능 출시" | "React 생태계를 신속하게 마스터하고 일정보다 앞서 프로덕션 기능 출시" |

**10단계 최적화 프로세스:**
1. **추출** -- 원시 정보 파싱, 각 데이터의 출처 기록
2. **보존** -- 모든 직무 경력 보존（결코 삭제하지 않음）
3. **계층화** -- 관련성으로 우선순위 지정: 핵심（3-5 하이라이트）/ 지원（2-3）/ 초기（1-2）
4. **정량화（진실한）** -- 3단계 전략: 정확한 데이터 보존 / 모호한 데이터는 추정으로 표시 / 데이터 없음은 정성적 서술로 변경
5. **귀속 잠금** -- 각 설명이 올바른 회사에 속하는지 확인
6. **재작성** -- 액션 동사와 STAR 방법론 사용
7. **재구성** -- 전문 이력서 형식
8. **HR 검증** -- 실제 채용 관리자의 심사 기준에 대해 자체 검증
9. **확인** -- 투명한 전후 비교 표시, 사용자 승인
10. **JD 매칭** -- 목표 직무 JD에서 키워드를 추출하고, 커버리지 갭을 분석하며, 매칭도 향상을 위한 최적화 제안 생성

**핵심 원칙:**
- **결코 데이터를 조작하지 않음** -- 제공되지 않은 숫자는 상상하지 않음
- **모든 경력 보존** -- 모든 직무 경력이 이력서에 남음
- **회사 귀속 잠금** -- 설명이 회사 간에 혼동되지 않음
- **콘텐츠 밀도 선택** -- 간결版（1 페이지）/ 표준版（1-2 페이지）/ 상세版（2-3 페이지）

**원본 콘텐츠를 유지하고 싶으신가요?** `/raw` 입력 -- 제로 수정, 순수 디자인 변환.

### 3. 완전한 출력 패키지

한 번의 호출로, **10가지 산출물**:

| 파일 | 용도 |
|------|------|
| `resume-{이름}-{스타일}.html` | 멋진 비주얼 이력서 -- 브라우저에서 열기 |
| `resume-{이름}-{스타일}.pdf` | **자동 낳은 PDF** -- 픽셀 완벽, 인쇄 가능 |
| `resume-{이름}-{스타일}.png` | **고해상도 PNG** -- 2x A4 스크린샷, 소셜 미디어나 메신저 공유에 최적 |
| `resume-{이름}-ats.html` | 일반 텍스트 ATS 버전 -- 지원자 추적 시스템에 안전 |
| `resume-{이름}.md` | Markdown 백업 -- 언제든지 편집 및 재사용 |
| `cover-letter-{이름}.html` | 매칭되는 자기소개서 -- 동일한 비주얼 스타일 |
| `linkedin-export.html` | LinkedIn 최적화 콘텐츠 -- 복사하여 바로 사용 |
| `job-search-kit.html` | **구직 전략 보고서** -- 목표 기업, 타임라인, 급여 참고, 연락 템플릿 |
| `interview-prep.html` | **면접 준비 보고서** -- 예상 질문과 답변 프레임워크, 기업 정보 |
| `resume-diagnosis.html` | **이력서 진단 보고서** -- JD 매칭 분석과 6차원 스코어링 |

**PDF는 자동으로 생성됩니다**: Playwright 헤드리스 브라우저를 사용하여 자동으로 낳습니다. 색상, 글꼴, 그라데이션, 레이아웃은 브라우저 표시와 완전히 일치합니다. Ctrl+P를 누를 필요가 없습니다.

**PNG 낳은** -- 2x A4 해상도의 고화질 PNG 스크린샷도 함께 생성, LinkedIn이나 WeChat 등 플랫폼에서 공유하기에 최적. Playwright를 사용할 수 없을 때는 PNG가 안정적인 폰백으로 작동합니다.

### 4. 이력서 진단 -- JD 매칭 분석

목표 직무 JD를 업로드하면 AI 진단 보고서를 받습니다:

- **종합 매칭 점수**（0-100）와 등급 평가（A/B/C）
- **6차원 세부 분석** -- 하드 스킬, 경험, 소프트 스킬, 학력, 키워드 커버리지, 이력서 품질
- **키워드 갭 분석** -- 어떤 JD 키워드가 커버되었는지 / 부족한지
- **AI 최적화 제안** -- 우선순위가 매겨진 실행 가능한 개선안（필수 / 권장 / 팁）

### 5. JSON 데이터 지속성

이력서 데이터는 구조화된 JSON으로 자동 저장:

- **재사용성** -- 과거 이력서 데이터를 로드하여 새로운 스타일을 생성하거나 콘텐츠를 업데이트할 수 있음
- **버전 관리** -- 각 생성에 타임스탬프와 버전 번호 포함
- **세션 간** -- 이력서 데이터는 대화 간에 지속되어 반복적인 개선이 용이

## 작업 흐름

```
1단계: 정보 제공              2단계: AI 최적화              3단계: 내용 확인
|-- 업로드 / 붙여넣기 / 대화   |-- 추출 및 모든 경력 보존     |-- 최적화 전후 비교
|-- 5가지 입력 방법            |-- 진실한 데이터 정량화       |-- 조정 또는 승인
|-- /raw로 원본 유지           |-- 전문적 표현 재작성          |-- 밀도 및 스타일 선택
                              |-- 회사 귀속 잠금
                              |-- JD 키워드 추출 및 갭 분석（JD 제공 시）

4단계: 스타일 선택             5단계: 모든 산출물 받기
|-- 29개 프리미엄 디자인        |-- 비주얼 이력서（HTML + PDF + PNG）
|-- AI가 최적의 3개 추천        |-- ATS 버전 + Markdown 백업
|-- 생성 전 미리보기             |-- 커버레터 + LinkedIn 콘텐츠
                              |-- 구직 전략 보고서
                              |-- 면접 준비 보고서
                              |-- 이력서 진단 보고서（JD 업로드 후）
                              |-- 향후 재사용용 JSON 데이터
```

**두 가지 모드：**
- **기본 모드** -- AI가 콘텐츠를 최적화（권장）
- **RAW 모드** -- 디자인만 변경, 콘텐츠 수정 없음（`/raw` 입력）

## 빠른 시작

### 사용자용

```bash
# 모든 AI 코딩 어시스턴트에서
use resume-design-pro
```

프롬프트에 따라:
1. 이력서 정보 제공 (업로드, 붙여넣기 또는 대화)
2. 콘텐츠 최적화 모드(기본값) 또는 RAW 모드 선택
3. 29가지 디자인 스타일 중 선택
4. (선택) 목표 JD를 업로드하여 이력서 진단 및 키워드 최적화 실행
5. 완전한 이력서 패키지 받기

## 플랫폼 호환성

**Resume Design Pro**는 오픈 SKILL 형식으로 구축되었습니다 -- Skill 호출을 지원하는 모든 AI 어시스턴트에서 작동합니다.

| 카테고리 | 예시 |
|----------|------|
| AI 코딩 어시스턴트 | Claude Code、Trae、Codex、Cursor、GitHub Copilot Chat |
| AI Agent 플랫폼 | Coze、Dify、LangChain Agents、AutoGen |
| 커스텀 AI 도구 | Markdown 기반 Skill 파일을 지원하는 모든 플랫폼 |

**요구사항:** 플랫폼은 다음만 필요합니다：
1. Markdown 지시사항 읽기 및 실행
2. 텍스트/HTML 출력 생성
3. 기본적인 사용자 상호작용 지원（프롬프트 → 응답）

특수 플러그인、API、통합이 필요하지 않습니다.
